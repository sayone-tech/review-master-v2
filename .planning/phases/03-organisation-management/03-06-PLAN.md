  <action>
    Under React 18 StrictMode the OrgManagement component mounts, unmounts, and remounts. `CreateButtonBridge` receives `onOpen={() => setCreateOpen(true)}` as a new inline function on every render, which causes its useEffect dependency array `[onOpen]` to re-run on every render — removing the click listener and immediately re-adding it, but the intermediate state (no listener) can coincide with the user's click, and StrictMode's double-invoke makes this worse.

    FIX — frontend/src/entrypoints/org-management.tsx

    Step 1. Update the import on line 1 to include `useCallback`:

        Before:
            import { useState, StrictMode, useEffect } from "react";
        After:
            import { useState, StrictMode, useEffect, useCallback } from "react";

    Step 2. Inside the `OrgManagement` component (between the useState hooks and the `return` statement — recommended around line 26 just above the `handleResendPlaceholder` declaration) add:

        const handleOpenCreate = useCallback(() => setCreateOpen(true), []);

    Step 3. Change the CreateButtonBridge usage on line 100 from:

        <CreateButtonBridge onOpen={() => setCreateOpen(true)} />

    to:

        <CreateButtonBridge onOpen={handleOpenCreate} />

    Do NOT modify the CreateButtonBridge component itself — its `[onOpen]` dependency array is correct. The fix is purely about giving it a stable reference.

    Do NOT change any other onClose/onCreated/onEdit closures. They are passed to modal components where the render-per-click-cost is acceptable; only CreateButtonBridge has the listener-registration race.
