## Stack Translation Map

The reference is pure React. The production stack per CLAUDE.md is Django templates + Tailwind CSS with React embedded only for complex interactive widgets.

| Reference Component | Production Target | Reason |
|---------------------|-------------------|--------|
| Shell (sidebar, topbar) | Django base template + Tailwind + Alpine.js | Static structure with minor interactivity (sidebar collapse, avatar dropdown) |
| Login page | Django template + Tailwind | Simple form, no complex state |
| Invitation acceptance page | Django template + Tailwind + Alpine.js | Form with password strength indicator (client-side only) |
| Profile page | Django template + Tailwind | Two simple forms |
| Organisations list (table, filter bar, search, pagination) | React widget | Complex client state, filter coordination, row actions |
| Row action three-dot menu | React (inside list widget) | Shares state with the table |
| Create / Edit Organisation modal | React widget | Form validation, submit lifecycle, integrated with list widget |
| Organisation Details modal | React widget | Rendered within the list widget's action flow |
| Adjust Store Count modal | React widget | Live validation against current in-use count |
| Confirmation popups | React component (reused across list widget) | Tight coupling to the action that triggered them |
| Toast notifications | Alpine.js on `<body>` seeded from Django messages; React widgets dispatch via custom event | Visible globally; works without React on non-widget pages |
| Status badges, buttons, inputs | Django template partials with Tailwind utility classes; matching React components inside widgets | Shared visual contract, two implementations of the same tokens |

**Integration contract:**
- React widgets mount via `django-vite` into Django templates at named `<div id="...">` mount points.
- Django emits initial data to widgets via a `<script type="application/json">` block.
- Toasts dispatched from React: `window.dispatchEvent(new CustomEvent('app:toast', { detail: { kind, title, msg } }))`.
- Alpine listener on `<body>` appends to the toast stack.
- All Tailwind classes used inside React widgets resolve against the single `tailwind.config.js` â€” no separate style pipelines.
