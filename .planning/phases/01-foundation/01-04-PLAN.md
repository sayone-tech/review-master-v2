    **Why this task includes a logout URL stub:**

    Templates `partials/sidebar.html` and `partials/topbar.html` use `{% url 'logout' %}` to point the logout `<form action>` at a resolvable URL. Phase 1 does NOT yet install `django.contrib.auth.urls` (auth flows are Phase 2). Without a stub, `{% url 'logout' %}` raises `NoReverseMatch` at render time, which breaks every test that hits any page (including `test_base_template.py`, `test_responsive.py`, `test_showcase.py`).

    The fix: register a minimal stub `/logout/` URL in `apps/common/urls.py` with `name='logout'`. The view accepts POST and returns a 302 to `/`. Phase 2 will replace the view body (not the URL name) with the real logout flow. The `name='logout'` identifier is stable across both phases so sidebar/topbar templates do not change.

    Step 1 — Create `apps/common/templatetags/__init__.py` (empty file) and `apps/common/templatetags/ui.py`:

    ```python
    from django import template
    from django.urls import reverse, NoReverseMatch

    register = template.Library()


    @register.simple_tag(takes_context=True)
    def is_active_route(context, prefix: str) -> bool:
        """Return True when the current request path starts with prefix.

        Used by sidebar nav items to apply the active state class bundle.
        """
        request = context.get("request")
        if request is None:
            return False
        return request.path.startswith(prefix)


    @register.simple_tag(takes_context=True)
    def is_active_url(context, url_name: str) -> bool:
        """Return True when the current request path matches a named URL."""
        request = context.get("request")
        if request is None:
            return False
        try:
            target = reverse(url_name)
        except NoReverseMatch:
            return False
        return request.path == target
    ```
