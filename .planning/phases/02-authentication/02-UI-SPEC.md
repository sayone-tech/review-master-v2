## Layout Contract

### auth_base.html structure

- `<body>` background: `bg-bg` (`#FAFAFA`) — full viewport, flex-centered
- Centered container: `max-w-[480px]`, `w-full`, `px-4` (horizontal gutters on narrow viewports)
- Card: `bg-white`, `rounded-card` (12px), `border border-line`, `shadow-sm`, `px-8 py-8`
- Logo block sits above the card heading, inside the card at the top
- Single scrollable column — no sidebar, no topbar

### Card dimensions

| Property | Value | Source |
|----------|-------|--------|
| Max width | 480px | CONTEXT.md — "~400–480px wide" |
| Min width | 320px (via px-4 on container) | default |

| Border radius | 12px (rounded-card) | tailwind.config.js |
| Shadow | `shadow-sm` | Claude's discretion — consistent with Phase 1 card style |

- App logo mark: `w-8 h-8 rounded-logo bg-yellow text-black font-extrabold text-[14px] flex items-center justify-center` — "R" glyph

## Typography

Source: Phase 1 design system — Geist font, established class patterns from form_fields.html and buttons.html.

Exactly 4 sizes and 2 weights are declared.

| Role | Size | Weight | Line Height | Usage in Phase 2 |
|------|------|--------|-------------|-----------------|
| Card heading | 20px (`text-[20px]`) | 600 (semibold) | 1.2 | "Sign in to your account", "Forgot your password?", "Set new password" |
| Subheading / card body | 14px (`text-sm`) | 400 (regular) | 1.5 | Descriptive text below card heading, email-sent body copy |
| Labels / inputs / buttons / links | 13px (`text-[13px]`) | 400 (regular) | 1.4 | Field labels, input values, button labels, "Forgot password?", "Back to sign in", checkbox label, "Remember me" label |
| Helper / error text | 12px (`text-[12px]`) | 400 (regular) | 1.4 | Field error messages, helper text, password strength label |

Weights in use: 400 (regular) + 600 (semibold). No other weights used in Phase 2.

Card headings use weight 600. All other text — labels, inputs, buttons, links, helpers, subheadings — uses weight 400.

### Password strength indicator

Required on the reset-password confirm page (`password_reset_confirm.html`) per CONTEXT.md. Alpine.js implementation consistent with Phase 1 patterns.

Strength levels and visual treatment:

| Score | Label | Bar color |
|-------|-------|-----------|
| 0–1 | Weak | `bg-red` |
| 2 | Fair | `bg-amber` |
| 3 | Good | `bg-yellow` |
| 4 | Strong | `bg-green` |

Bar: 4 segments, `h-1`, `rounded-full`, `w-full`, spaced by `gap-1`. Rendered below the password input, above helper text. Label text: `text-[12px] text-subtle`, right-aligned to bar.

Strength logic (Alpine.js, inline): checks length >= 8, uppercase present, number present, special char present — each criterion adds 1 point.

## Form Layouts

### Login page (`login.html`)

Vertical stack inside the card:

1. Logo block (`mb-6`)
2. Heading: "Sign in to your account" (`text-[20px] font-semibold text-ink mb-1`)
3. Subheading: "Enter your credentials to access the admin panel." (`text-sm text-subtle mb-6`)
4. Error alert banner (conditional, renders when Django form has non-field errors — `mb-4`)
5. Email field (`mb-4`) — label "Email address", type=email, autocomplete=email
6. Password field row:
   - Label row: `flex items-center justify-between mb-2` — "Password" on left, "Forgot password?" link on right
   - Password input (no label wrapper — label is in the row above)
   - Field error beneath input (`mt-2`)
7. Remember me checkbox (`mt-2 mb-4`)
8. Primary CTA button: full-width, "Sign in" (`w-full mt-2`)
9. Rate-limit message renders as a top-of-form alert banner (same style as login error) when 429 is returned

"Forgot password?" link: `text-[13px] text-blue hover:underline`, `href="/password-reset/"`.

### Forgot password page (`password_reset.html`)

1. Logo block (`mb-6`)
2. Heading: "Forgot your password?" (`text-[20px] font-semibold text-ink mb-1`)
3. Subheading: "Enter your email and we'll send you a reset link." (`text-sm text-subtle mb-6`)
4. Email field — label "Email address", type=email
5. Primary CTA button: full-width, "Send reset link" (`w-full mt-2`)
6. Back link: centered below button, "Back to sign in" — `text-[13px] text-blue hover:underline`, `href="/login/"`, `mt-4 block text-center`

### Email sent confirmation page (`password_reset_done.html`)

No form. Informational card only:

1. Logo block (`mb-6`)
2. Check icon: `w-12 h-12 rounded-full bg-green-tint text-green flex items-center justify-center mx-auto mb-4` — Lucide `mail-check`, 24px
3. Heading: "Check your email" (`text-[20px] font-semibold text-ink mb-2 text-center`)
4. Body: "If an account exists for that email address, you'll receive a password reset link within a few minutes." (`text-sm text-subtle text-center mb-6`)
5. Back link: centered, "Back to sign in"

### Reset password page (`password_reset_confirm.html`)

1. Logo block (`mb-6`)
2. Heading: "Set a new password" (`text-[20px] font-semibold text-ink mb-1`)
3. Subheading: "Your new password must be at least 10 characters." (`text-sm text-subtle mb-6`)
4. New password field (type=password, with show/hide toggle)
5. Strength indicator (below new password field, above confirm field)
6. Confirm password field (type=password, with show/hide toggle, label "Confirm new password")
7. Primary CTA button: full-width, "Set new password"

Invalid/expired token state: full-card error message, no form rendered:
- Heading: "This link has expired"
- Body: "Password reset links expire after 1 hour and can only be used once. Please request a new one."
- Link: "Request a new reset link" → `/password-reset/`

## Interaction States

### Sign in button — loading state

When the login form is submitted, the button enters a loading state (Alpine.js `x-data="{loading: false}"` on the `<form>`, `@submit="loading = true"` on the form, `:disabled="loading"` on the button).

Button in loading state:
- Replace label text with spinner: `inline-block w-[14px] h-[14px] border-2 border-black/20 border-t-black rounded-full animate-spin`
- Button text: "Signing in…"
- Button remains visually yellow (not disabled-dimmed) — only pointer-events disabled

Source: CONTEXT.md — "Loading / spinner state on the sign-in button during form submission" is Claude's discretion.

### Form field focus

Established in Phase 1 form_fields.html: `focus:ring focus:ring-black/[0.06] focus:border-ink`. All auth form inputs inherit this.

### Password show/hide toggle

Established in Phase 1 form_fields.html. Alpine.js `x-data="{ show: false }"`, `:type="show ? 'text' : 'password'"`. Lucide `eye` / `eye-off` icons swap on toggle. Toggle button is `w-8 h-8` (32px square). Button is `tabindex="-1"` to not interrupt form tab order.

## Accessibility Contract

Inherits all Phase 1 WCAG AA decisions (DSYS-08). Additional Phase 2 specifics:

| Requirement | Implementation |
|-------------|---------------|
| Error banner announced to screen readers | `role="alert"` on the error banner div |
| Field errors announced | `role="alert"` on field error `<p>` (established in form_fields.html) |
| Auth card has landmark | `<main>` wrapping the card in auth_base.html |
| Heading hierarchy | `<h1>` for card heading; `<h2>` not needed at this scale |
| Form autocomplete | `autocomplete="email"` on email field, `autocomplete="current-password"` on login password, `autocomplete="new-password"` on reset/confirm password fields |
| Focus trap | Not required — auth pages have no modal dialogs |
| Remember me checkbox | Properly associated label (`<label>` wraps `<input>`) |
| Strength indicator | `aria-live="polite"` region for screen readers |
